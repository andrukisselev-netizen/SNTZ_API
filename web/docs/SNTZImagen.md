# SNTZimage

Генерация изображений по тексту (и текст + картинки) через SNTZ_API. Модели Gemini (gemini-2.5, gemini-3.1).

---

## Перед началом работы

**Настройки ComfyUI:**
1. **Settings** (⚙️) → **Comfy** → **Locale** → **Language** → **English**
2. **Settings** (⚙️) → **Experimental** → **Modern Node Design (Nodes 2.0)** — включить

**Общие напоминания:**
- ✓ VPN выключен
- ✓ API-ключ вставляется через Photoshop и один раз в каждую ноду

---

## Как использовать SNTZimage

1. **Выберите** соотношение сторон (`aspect_ratio`) и модель (`model`).
2. При необходимости отключите **use_image_url_delivery** (по умолчанию включён — ответ со ссылками на файлы при настроенном сервере).
3. Введите **prompt** и запустите Queue Prompt.

Подробные логи запроса в консоль ComfyUI включены всегда (параметр **debug** в интерфейс не выводится).

---

## Параметры

| Параметр | Описание |
|----------|----------|
| **prompt** | Текст описания изображения |
| **model** | gemini-2.5 (до 3 входных изображений), gemini-3.1 (до 5) |
| **api_key** | API-ключ (пусто = из .api_key) |
| **aspect_ratio** | 1:1, 16:9, 9:16, 4:3, 3:4 и др. |
| **resolution** | 1K, 2K, 4K |
| **use_image_url_delivery** | **true** по умолчанию — запрос с заголовком `X-SNTZ-Image-Delivery: url`: сервер кладёт PNG в `/gen/` и возвращает ссылку вместо base64 (нужна настройка на стороне New API + Nginx). См. [GEMINI_IMAGE_URL_DEPLOY.md](../../GEMINI_IMAGE_URL_DEPLOY.md). Выключите, если сервер отдаёт только inline base64. |

Альтернатива: переменная окружения ComfyUI `SNTZ_IMAGE_DELIVERY_URL=1` (включает тот же режим).

---

## Выходы ноды

| Выход | Описание |
|-------|----------|
| **images** | Тензор изображения (или служебный превью 64×64, если картинка по URL не скачалась). |
| **credits** | Баланс и сведения по ключу. |
| **image_urls** | HTTP(S)-ссылки на файлы на сервере (по одной на строку), если API их вернул. Если сервер ответил только встроенным base64, здесь будет **поясняющий текст** (как включить `GEMINI_IMAGE_*` и `/gen/`). В консоли Comfy при реальных ссылках печатается строка `image_url: …` на каждый URL. |

## Ограничения и рекомендации

- Не загружайте изображения разрешением больше 4K (следите за разрешением и DPI).
- Загружайте только JPEG и PNG.
- Желательно соблюдать соотношения сторон, поддерживаемые моделью (1:1, 16:9, 9:16, 4:3, 3:4 и др.).
- Модель может некорректно обрабатывать цветовые профили. Старайтесь использовать sRGB.

---

## Ошибки

При возникновении ошибок попробуйте несколько раз. Проверьте VPN, API-ключ, переключите модель. При повторении ошибки более 3 раз подряд обратитесь к системному администратору или в техподдержку.
